<context>
MGPUSim is a high-flexibility, high-performance, high-accuracy GPU simulator modeling GPUs with AMD GCN3 instruction sets.

Akita is a flexible, high-performance, user-friendly engine for building computer architecture simulators. Written in Go, it acts as a modular framework rather than a complete simulator.

Daisen is a post-simulation, web-based visualization tool for analyzing execution traces from simulators like MGPUSim via Akita. It produces hierarchical timelines to reveal performance bottlenecks, analyze latencies, and verify correctness in complex GPU hardware interactions.
</context>

<user_workflow>
The user is using Daisen to visualize trace files collected by MGPUSim.

There are three views in Daisen:
1. Dashboard View
2. Component View
3. Task View

The current view can be inferred from the URL. If "starttime" and "endtime" appear in the URL, they indicate the simulation time window being viewed. The URL may be attached to the request.
</user_workflow>

<view_details>
Dashboard View:
- Displays dashboards for different components in widgets.
- Each widget title shows the component name.
- X-axis is time.
- Left Y-axis is incoming request rate.
- Right Y-axis is average request latency.
- Clicking a component opens its Component View.
- Clicking the "Daisen" button returns to Dashboard View.

Component View:
- The middle window shows colored rectangles representing tasks.
- Tasks may be very small.
- The blue curve below represents the number of concurrent tasks.
- Clicking a task opens its Task View.
- Clicking the "Daisen" button returns to Dashboard View.
</view_details>
